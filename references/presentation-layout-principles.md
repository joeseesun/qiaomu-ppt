# Presentation Layout Principles

Use this reference before choosing concrete patterns from `layout-pattern-library.md`.
It defines the reading and hierarchy rules that make a slide feel intentional
instead of template-filled.

## Reading Path

- Pick one primary landing point per slide: title, proof object, number, quote,
  or product image.
- Arrange secondary objects so the eye can move in one obvious path:
  left-to-right, top-to-bottom, center-out, object-to-annotation, or
  step-by-step.
- Do not create two unrelated first reads. If the title and chart compete,
  shrink or quiet one of them.
- Align evidence and explanation physically. A takeaway belongs near the chart,
  screenshot, image, or process step it explains.

## Hierarchy

- Use scale before decoration. The main claim should usually be the largest
  text element on normal argument slides.
- Use weight, contrast, and color to clarify importance, not to decorate every
  object.
- Give multi-line Chinese/CJK titles enough leading. A title can be bold and
  cinematic without being crushed: normal title line-height should sit around
  1.14-1.30, while very large cover titles should not go below 1.06 unless the
  line is a one-line wordmark.
- Leave whitespace around the dominant object. Crowded slides make the audience
  search instead of understand.
- Keep body copy in short chunks. When text needs paragraph depth, split the
  slide or move nuance into speaker notes.

## Grouping

- Group by proximity and alignment first. Add cards, outlines, dividers, or
  shadows only when whitespace cannot carry the relationship.
- Use repeated cards only for true peer items. A slide with one main proof and
  three small notes should not pretend all four objects are equal.
- Avoid card-inside-card layouts. They add visual chrome without adding meaning.
- A connector means sequence, causality, hierarchy, dependency, or flow. If it
  does not express one of those relationships, remove it.

## Evidence Fit

- Match the layout to the proof job:
  comparison, trend, composition, distribution, relationship, process,
  architecture, concept, objection, or decision.
- Give charts and screenshots enough area to be inspectable. A tiny proof object
  is usually a sign that the slide is trying to do too much.
- For source images, declare the crop or containment choice before rendering.
  Do not rely on a fake rounded frame to hide overflow.

## Rhythm

- Alternate dense, anchor, and breathing pages in long decks so the audience can
  reset between evidence-heavy slides.
- Reuse a visual system, not the same composition. Long decks need multiple
  layout families even when the palette and typography stay consistent.
- Courseware may need denser pages, but the learning state should still advance
  one concept or exercise at a time.
